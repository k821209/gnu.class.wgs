📜 Raw Data에서 Variant까지의 여정

WGS 분석은 원본 시퀀싱 데이터(FASTQ)에서 시작하여 생물학적 의미를 가지는 변이(Variant)를 찾아내는 다단계 과정입니다. 각 단계는 다음 단계의 입력 데이터 품질을 결정하므로 매우 중요합니다.

Raw Data (FASTQ)

시퀀서에서 생성된 원본 데이터

➡️

Quality Control (QC)

데이터 품질 평가 및 진단

➡️

Pre-processing

불필요한 서열 제거 및 필터링

➡️

Alignment

참조 게놈에 리드 정렬

➡️

Post-processing

정렬된 데이터 보정 및 정리

➡️

Variant Calling

유전적 변이 식별

🔍 각 단계별 목적

  • Raw Data (FASTQ): 시퀀서가 생성한 원본 염기 서열 및 품질 정보.
  • Quality Control (QC): 원본 데이터의 품질을 평가하고 문제점을 진단. (e.g., FastQC)
  • Pre-processing: 어댑터 서열, 낮은 품질의 염기 등 불필요한 부분을 제거. (e.g., Trimmomatic)
  • Alignment: 처리된 리드를 참조 게놈(Reference Genome)에 정렬. (e.g., BWA-MEM)
  • Post-processing: 정렬된 데이터(BAM)에서 중복 리드를 제거하고 정렬 품질을 보정. (e.g., SAMtools, Picard)
  • Variant Calling: 참조 게놈과의 차이(변이)를 통계적으로 식별. (e.g., GATK HaplotypeCaller)

🐍 Conda 환경 설정 및 도구 설치

WGS 분석에 필요한 모든 도구들을 Conda를 통해 체계적으로 관리하고 설치하는 방법을 알아봅시다.

📦 1. Conda 환경 생성

분석 도구들 간의 의존성 충돌을 방지하기 위해 전용 환경을 생성합니다.

# WGS 분석 전용 환경 생성 conda create -n wgs-analysis python=3.9 conda activate wgs-analysis # Bioconda 채널 추가 (생물정보학 도구들) conda config --add channels bioconda conda config --add channels conda-forge

🔧 2. 품질 평가 도구 설치

# Short Read 품질 평가 conda install fastqc # Long Read 품질 평가 conda install nanoplot pip install longqc # MultiQC (여러 QC 결과 통합) conda install multiqc

✂️ 3. 전처리 도구 설치

# Short Read 전처리 conda install trimmomatic conda install fastp # Long Read 전처리 conda install porechop conda install filtlong

🗃️ 4. 정렬 및 후처리 도구 설치

# 정렬 도구 conda install bwa conda install hisat2 conda install minimap2 # Long read 정렬 # SAM/BAM 처리 conda install samtools conda install picard # 중복 제거 및 정렬 통계 conda install bamtools

🔍 5. Variant Calling 도구 설치

# GATK4 (주요 variant caller) conda install gatk4 # 대안 variant callers conda install freebayes conda install bcftools # VCF 파일 처리 conda install vcftools conda install bedtools

📊 6. 분석 및 시각화 도구

# Python 분석 라이브러리 conda install pandas numpy matplotlib seaborn conda install jupyter # R 및 Bioconductor (선택적) conda install r-base r-ggplot2 conda install bioconductor-genomicranges

💡 환경 관리 팁

  • 환경 활성화: conda activate wgs-analysis
  • 환경 비활성화: conda deactivate
  • 설치된 패키지 확인: conda list
  • 환경 내보내기: conda env export > environment.yml
  • 환경 복원: conda env create -f environment.yml
