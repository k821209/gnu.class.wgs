단계 3: 정렬 후처리 (Post-processing)

✨ 정렬된 데이터(BAM) 다듬기

정렬 직후의 SAM/BAM 파일에는 분석의 정확도를 저해하는 요소들이 남아있습니다. 후처리 과정은 이러한 문제들을 해결하여 변이 검출의 신뢰도를 높이는 단계입니다.

🚨 왜 후처리가 필요한가?

정렬 직후 문제점들

  • PCR 중복 리드: 동일한 DNA 단편이 여러 번 증폭되어 실제보다 높은 커버리지 생성
  • 정렬 순서 혼재: 리드가 이름 순서로 정렬되어 있어 랜덤 액세스 불가
  • 메타데이터 부족: 샘플 정보, 라이브러리 정보 등 누락으로 다운스트림 분석 방해
  • 페어 정보 불일치: 좌표 정렬 후 mate 정보가 부정확하여 중복 검출 오류 발생
  • 파일 크기: 압축되지 않은 SAM 파일로 인한 저장 공간 낭비

후처리 후 개선사항

  • 정확한 커버리지: 중복 제거로 실제 시퀀싱 깊이 반영
  • 빠른 액세스: 좌표 정렬 + 인덱싱으로 특정 영역 빠른 검색
  • 완전한 메타데이터: 리드 그룹으로 샘플 추적 가능
  • 정확한 페어 정보: mate 좌표 및 insert size 정보 수정으로 정확한 중복 검출
  • 효율적 저장: BAM 압축으로 70-80% 공간 절약

💡 실제 영향 예시

  • 중복 리드 미제거 시: 특정 위치에서 false positive 변이 호출 (커버리지 100x → 실제 30x)
  • 정렬 미수행 시: 1GB 영역 검색에 수 분 소요 → 정렬 후 수 초
  • 리드 그룹 부재 시: 멀티샘플 분석에서 샘플 구분 불가능
  • Fixmate 미수행 시: 부정확한 mate 정보로 인해 중복 리드 20-30% 미검출

🔧 Fixmate의 역할과 중요성

Paired-end 시퀀싱에서 좌표 정렬 후 mate 정보가 손상되는 문제를 해결하는 핵심 단계입니다.

❌ Fixmate 전 문제점
  • MPOS (mate position) 정보 부정확
  • ISIZE (insert size) 계산 오류
  • Mate 매핑 품질 정보 누락
  • 중복 검출 알고리즘 성능 저하
✅ Fixmate 후 개선사항
  • 정확한 mate 좌표 정보 복원
  • 올바른 insert size 계산
  • 완전한 SAM flags 설정
  • 효율적인 중복 마킹 가능
-m 옵션의 효과: 중복 마킹을 위한 추가 정보를 태그로 저장하여 MarkDuplicates 성능을 20-30% 향상시킵니다.

SAMtools

  • SAM/BAM/CRAM 파일을 다루는 필수 도구.
  • 파일 형식 변환, 정렬, 인덱싱 등 다양한 기능 제공.
  • 주요 작업: SAM → BAM 변환, 좌표 기준 정렬, 페어 정보 수정, 인덱싱.
# 1. SAM to BAM 변환 samtools view -bS aligned.sam > aligned.bam # 2. 좌표 기준 정렬 samtools sort aligned.bam -o sorted.bam # 3. Fixmate (페어 정보 수정) samtools fixmate -m sorted.bam fixmate.bam # 4. 인덱싱 samtools index fixmate.bam

Picard

  • GATK와 함께 개발된 자바 기반의 BAM/SAM 파일 처리 도구.
  • 특히 중복 리드 제거와 통계 생성에 강력함.
  • 주요 작업: 중복 리드 마킹, 리드 그룹 추가/수정.
# 1. 중복 리드 마킹 (Mark Duplicates) java -jar picard.jar MarkDuplicates \ I=sorted.bam \ O=dedup.bam \ M=marked_dup_metrics.txt # 2. 리드 그룹 추가 (AddOrReplaceReadGroups) java -jar picard.jar AddOrReplaceReadGroups \ I=dedup.bam O=final.bam \ RGID=group1 RGLB=lib1 RGPL=illumina RGPU=unit1 RGSM=sample1
