파라미터 선택 기준

⚙️ 분석 단계별 파라미터 최적화

WGS 분석의 각 단계에서 사용되는 도구들은 다양한 파라미터를 가집니다. 이 파라미터들은 데이터의 특성, 분석 목표, 컴퓨팅 자원 등에 따라 신중하게 선택되어야 합니다.

🗑️ 전처리 (Trimmomatic, fastp)

품질 필터링

  • Q 값 (품질 점수): 일반적으로 Q20 또는 Q30을 기준으로 설정.
  • 슬라이딩 윈도우: 4bp 윈도우에 평균 Q15 이상 (Trimmomatic).
  • 최소 길이: 36bp 이상 (짧은 리드는 정렬 정확도 저하).

어댑터 제거

  • 어댑터 서열: 시퀀싱 플랫폼(Illumina, MGI 등)에 맞는 어댑터 파일 사용.
  • 오버랩 길이: 어댑터와 리드 간 최소 오버랩 길이 설정 (예: 10bp).
  • 에러 허용치: 어댑터 매칭 시 허용할 에러 수 (예: 2).

🗺️ 정렬 (BWA-MEM, HISAT2)

스레드 수 (-t)

  • 사용 가능한 CPU 코어 수에 맞춰 설정 (예: -t 8).
  • 병렬 처리로 정렬 속도 향상.

참조 게놈

  • 분석 대상 생물종의 최신 및 고품질 참조 게놈 사용.
  • 인덱싱 과정 필수 (BWA index, hisat2-build).

✨ 후처리 (SAMtools, Picard)

정렬 및 인덱싱

  • 좌표 기준 정렬: `samtools sort` (변이 검출 전 필수).
  • 인덱싱: `samtools index` (빠른 접근을 위해).

중복 리드 제거

  • MarkDuplicates (Picard): PCR 증폭으로 인한 인위적 중복 제거.
  • 메트릭스 파일: 중복률 확인 및 분석 품질 평가.

💎 변이 검출 (GATK, FreeBayes)

GATK HaplotypeCaller

  • 참조 게놈: 정렬에 사용된 것과 동일한 참조 게놈.
  • 리드 그룹: Picard로 추가된 리드 그룹 정보 활용.
  • GVCF 모드: 여러 샘플 통합 분석 시 gVCF 생성.

FreeBayes

  • 배수성 (-p): 생물종의 배수성 설정 (예: 인간은 -p 2).
  • 최소 커버리지: 변이 호출에 필요한 최소 리드 수.
  • 최소 품질: 변이 호출에 필요한 최소 품질 점수.

💡 파라미터 선택 시 고려사항

  • 데이터 품질: 원본 데이터의 품질이 낮으면 필터링 파라미터를 더 엄격하게.
  • 분석 목표: 희귀 변이 탐지 시 더 높은 커버리지와 민감한 파라미터.
  • 생물종 특성: 게놈 크기, 반복서열, 배수성 등을 고려하여 도구 및 파라미터 선택.
  • 컴퓨팅 자원: CPU, 메모리, 저장 공간을 고려하여 스레드 수 등 설정.
  • Best Practices: 각 도구의 공식 문서나 커뮤니티에서 권장하는 Best Practices 참고.
