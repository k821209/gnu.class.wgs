DB 및 분석 환경

🌐 공개 데이터베이스 활용

생물정보학 분석을 위해서는 다양한 공개 데이터베이스를 활용하는 것이 필수적입니다. 특히 시퀀싱 데이터와 참조 게놈 정보는 분석의 시작점이 됩니다.

SRA/ENA 데이터 활용

  • SRA (Sequence Read Archive): NCBI에서 운영하는 시퀀싱 데이터 저장소.
  • ENA (European Nucleotide Archive): EMBL-EBI에서 운영하는 시퀀싱 데이터 저장소.
  • 활용: 공개된 시퀀싱 데이터를 다운로드하여 재분석 또는 비교 분석.
  • 팁: SRA-toolkit을 사용하여 데이터 다운로드 및 포맷 변환.

참조게놈 선택

  • Ensembl: 유전자 주석이 잘 되어 있는 동물 게놈 데이터베이스.
  • RefSeq: NCBI에서 제공하는 고품질의 참조 서열 데이터베이스.
  • 활용: 분석하고자 하는 생물종의 가장 최신 및 고품질 참조 게놈 선택.
  • 팁: 게놈 버전 확인 및 인덱싱 과정 필수.

🌱 식물 특화 게놈 데이터베이스

식물 게놈 분석을 위한 전문 데이터베이스들은 식물 특성에 맞춘 주석 정보와 분석 도구를 제공합니다.

Ensembl Plants

  • URL: plants.ensembl.org
  • 특징: 50+ 식물 종 게놈, 비교유전체학 도구
  • 제공 데이터: 유전자 주석, 변이 정보, 계통 분석
  • 장점: 표준화된 인터페이스, REST API 지원

Phytozome

  • URL: phytozome-next.jgi.doe.gov
  • 특징: JGI의 식물 게놈 포털, 고품질 게놈
  • 제공 데이터: 게놈 어셈블리, 유전자 패밀리, 비교 분석
  • 장점: 큐레이션된 데이터, 다운로드 편의성

PlantGDB

  • URL: plantgdb.org
  • 특징: 식물 EST/유전자 데이터베이스
  • 제공 데이터: 전사체 정보, 유전자 구조, 발현 분석
  • 장점: 전사체 중심 분석, 유전자 예측 도구

Sol Genomics Network

  • URL: solgenomics.net
  • 특징: 가지과 식물 특화 데이터베이스
  • 제공 데이터: 토마토, 감자, 후추 등 게놈 정보
  • 장점: 육종 정보, QTL 데이터, 표현형 정보

Gramene

  • URL: gramene.org
  • 특징: 벼과 식물 특화, 비교유전체학
  • 제공 데이터: 벼, 밀, 옥수수 등 주요 곡물 게놈
  • 장점: 진화 분석, 오솔로지 정보, 경로 분석

Plant Reactome

  • URL: plantreactome.gramene.org
  • 특징: 식물 대사 경로 데이터베이스
  • 제공 데이터: 생화학 경로, 단백질 상호작용
  • 장점: 경로 분석, 기능 주석, 시각화 도구

KEGG

  • URL: kegg.jp
  • 특징: 대사 경로, 유전자 온톨로지, 시스템 생물학
  • 제공 데이터: 경로 맵, 효소 정보, 게놈 비교
  • 장점: 표준화된 경로 분석, 식물 특화 경로 포함

WikiPathways

  • URL: wikipathways.org
  • 특징: 커뮤니티 기반 경로 데이터베이스
  • 제공 데이터: 생물학적 경로, 분자 상호작용
  • 장점: 오픈 소스, 식물 경로 풍부, 편집 가능

NCBI dbSNP

  • URL: ncbi.nlm.nih.gov/snp
  • 특징: 전 세계 최대 SNP/변이 데이터베이스
  • 제공 데이터: SNP, INDEL, 구조변이, 빈도 정보
  • 장점: 표준 rs ID, 식물 변이 포함, API 지원

NCBI ClinVar

  • URL: ncbi.nlm.nih.gov/clinvar
  • 특징: 임상적 의미를 가진 변이 정보
  • 제공 데이터: 변이-표현형 연관성, 병원성 분류
  • 장점: 큐레이션된 변이 효과, 기능 주석

🌾 식물 연구에서의 dbSNP 활용 관점

식물 게놈학에서 dbSNP의 특별한 고려사항

장점 및 활용

  • 품종/계통 비교: 작물 품종 간 SNP 차이 분석
  • 진화 연구: 야생종-재배종 간 변이 패턴
  • 육종 마커: 농업 형질 연관 SNP 식별
  • 집단 구조: 식물 집단의 유전적 다양성
  • 연관 분석: GWAS 연구의 reference SNP

제한사항 및 주의점

  • 데이터 편향: 주요 작물 중심, 야생종 부족
  • 다배체 복잡성: 밀, 면화 등 복잡한 게놈
  • 구조변이 한계: 대형 구조변이 정보 부족
  • 집단 대표성: 특정 지역/품종 편중
  • 기능 주석: 식물 특이 기능 정보 제한
실무 팁
  • 식물별 접근: 종별 SNP 밀도와 분포 패턴 확인 필수
  • 보완적 활용: 종별 전문 DB (TAIR, MaizeGDB)와 함께 사용
  • 품질 필터링: 식물 샘플 크기와 검증 수준 체크
  • 좌표 변환: 게놈 버전 간 좌표 매핑 주의

💻 분석 환경 구축

대용량 게놈 데이터를 효율적으로 분석하기 위해서는 적절한 컴퓨팅 환경을 이해하고 활용하는 것이 중요합니다.

농진청 슈퍼컴퓨터 2호기

  • 시스템: Cray XC40, 2,904개 계산노드
  • 노드 사양: Intel Haswell 2.6GHz, 24cores, 128GB RAM
  • 접속: VPN(평일 9-18시) 또는 SSH (172.29.110.61)
  • 스케줄러: PBS Pro + ALPS (Cray 특화)
📋 기본 PBS 명령어:
• qsub job.sh - 작업 제출
• qstat -u $USER - 내 작업 확인
• qdel jobid - 작업 취소
• qstat -Q - 큐 상태 확인

클라우드 플랫폼 소개

  • AWS, Google Cloud, Azure: 유연한 컴퓨팅 자원 제공.
  • 장점: 필요에 따라 자원 확장/축소, 초기 투자 비용 없음.
  • 활용: 가상 머신(VM) 생성, 스토리지 관리, 컨테이너(Docker) 활용.
  • 팁: 비용 관리 및 보안 설정에 유의.

🖥️ 농진청 슈퍼컴퓨터 실무 활용 가이드

PBS + ALPS 기반 식물 게놈 분석

PBS 작업 스크립트 (GATK 예시)

#!/bin/sh
#PBS -N gatk_variant_calling
#PBS -q nabic
#PBS -l select=1:ncpus=8:mpiprocs=1:ompthreads=8
#PBS -l walltime=24:00:00
#PBS -j oe

cd $PBS_O_WORKDIR
module use /s3/opt/modulefiles/
module load java/22

aprun -n 1 -d 8 /s3/opt/soft/gatk/gatk \
  HaplotypeCaller \
  -R reference.fa \
  -I aligned.bam \
  -O variants.vcf
스크립트 문법 해설
📝 PBS 지시문 (# 뒤에 PBS로 시작):
• #!/bin/sh - 쉘 지정 (sh 사용)
• #PBS -N - 작업 이름 설정
• #PBS -q - 사용할 큐 지정
• #PBS -l - 자원 요청 (select, walltime 등)
• #PBS -j oe - 표준출력/에러 통합

🔧 자원 할당 문법:
• select=1 - 노드 1개 사용
• ncpus=8 - 노드당 8개 코어
• mpiprocs=1 - 노드당 1개 MPI 프로세스
• ompthreads=8 - 프로세스당 8개 스레드

⚡ ALPS 실행 문법:
• aprun -n 1 - 총 프로세스 수 (select × mpiprocs)
• -d 8 - 프로세스당 스레드 수 (ompthreads와 일치)

큐별 자원 할당

  • nabic: 기본 계산노드용 큐
  • edu: 교육용 큐 (실습 권장)
  • nabic_m: 전후처리용 큐
  • 최대 노드: 100노드 (초과시 센터 문의)
  • 노드당 최대: 24cores, 128GB RAM

스토리지 및 파일 관리

  • 홈 디렉토리: /s1/home/rda/<account>
  • Lustre 파일시스템: /s1, /s3 사용
  • 파일 전송: SFTP, WinSCP, MobaXterm
  • 대용량 데이터: 외장하드 배송 가능
  • 인터넷 다운로드: 센터에 요청 (보안)

모듈 및 소프트웨어

  • 모듈 로드: module use /s3/opt/modulefiles/
  • 소프트웨어 설치: 센터에 요청
  • Singularity: v2.6.1 (Ubuntu 24.04 LTS)
  • 필수 명령: aprun 사용 (Cray 특화)
  • 프로그램 목록: /s3/program_list/program_list.txt
식물 게놈 분석 팁
  • 다배체 분석: select=1:ncpus=24:mpiprocs=1:ompthreads=24
  • 대용량 어셈블리: select=4:ncpus=24:mpiprocs=6:ompthreads=4
  • 병렬 처리: aprun -n 72 (select × mpiprocs)
  • I/O 집약적 작업: 센터와 사전 상의
