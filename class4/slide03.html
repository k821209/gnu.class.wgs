🔧 품질 관리

📊 NGS 데이터 품질 평가의 중요성

Raw 시퀀싱 데이터는 다양한 오류와 편향을 포함할 수 있습니다. 정확한 분석을 위해서는 데이터 품질을 평가하고 필요시 전처리가 필수입니다.

🎯 품질 관리 목표

주요 목표

  • 낮은 품질 염기 제거: Phred score < 20인 염기
  • 어댑터 시퀀스 제거: 라이브러리 준비 과정의 인공 서열
  • 오염 서열 제거: 다른 생물종 또는 벡터 서열
  • 길이 필터링: 너무 짧은 리드 제거

주요 문제점

  • 품질 저하: 시퀀싱 후반부 품질 감소
  • 어댑터 오염: 짧은 insert size로 인한 어댑터 읽기
  • PCR 중복: 라이브러리 증폭 과정의 편향
  • GC 편향: GC 함량에 따른 커버리지 편향

🔍 FastQC를 이용한 품질 평가

FastQC는 NGS 데이터의 품질을 종합적으로 평가하는 표준 도구입니다.

📈 주요 품질 지표

좋은 품질

  • Per base quality: 평균 Q30 이상
  • Per sequence quality: 대부분 Q30 이상
  • GC content: 이론값과 유사
  • Adapter content: 0% 또는 매우 낮음

주의 필요

  • Per base quality: 후반부 Q20-30
  • GC content: 약간의 편향
  • Sequence duplication: 중간 수준
  • Overrepresented seq: 소량 검출

나쁜 품질

  • Per base quality: Q20 미만
  • Adapter content: 높은 비율
  • Sequence duplication: 매우 높음
  • Per tile quality: 타일별 편향

💻 FastQC 실행

1단계: FastQC 설치

# FastQC 설치 (conda 사용) conda install -c bioconda fastqc # 또는 직접 다운로드 wget https://www.bioinformatics.babraham.ac.uk/projects/fastqc/fastqc_v0.12.1.zip unzip fastqc_v0.12.1.zip chmod +x FastQC/fastqc

2단계: FastQC 실행

# 단일 FASTQ 파일 분석 fastqc sample_R1.fastq.gz # 여러 파일 동시 분석 fastqc *.fastq.gz # 출력 디렉토리 지정 fastqc -o output_dir/ sample_R1.fastq.gz sample_R2.fastq.gz # 스레드 수 지정 (병렬 처리) fastqc -t 4 *.fastq.gz

✂️ Trimmomatic을 이용한 전처리

Trimmomatic은 어댑터 제거, 품질 필터링, 길이 필터링을 수행하는 강력한 도구입니다.

🛠️ 주요 기능

트리밍 기능

  • ILLUMINACLIP: 어댑터 시퀀스 제거
  • SLIDINGWINDOW: 슬라이딩 윈도우 품질 필터링
  • LEADING/TRAILING: 앞/뒤 낮은 품질 염기 제거
  • MINLEN: 최소 길이 필터링

처리 옵션

  • PE 모드: Paired-end 리드 처리
  • SE 모드: Single-end 리드 처리
  • 멀티스레딩: 병렬 처리 지원
  • 통계 보고: 처리 결과 요약

🔧 Trimmomatic 필터링 매개변수 상세 설명

필터링 옵션 완벽 가이드

ILLUMINACLIP 상세 매개변수

ILLUMINACLIP:<fastaWithAdaptersEtc>:<seed mismatches>:<palindrome clip threshold>:<simple clip threshold>[:<minAdapterLength>][:<keepBothReads>]
  • fastaWithAdaptersEtc: 어댑터 서열이 포함된 FASTA 파일
  • seed mismatches: 초기 매칭에서 허용할 최대 불일치 수 (보통 2)
  • palindrome clip threshold: 팰린드롬 모드 임계값 (30 권장)
  • simple clip threshold: 단순 모드 임계값 (10 권장)
  • minAdapterLength: 최소 어댑터 길이 (옵션, 기본값 8bp)
  • keepBothReads: true시 한쪽만 트리밍되어도 페어 유지
💡 팰린드롬 vs 단순 모드:
• 팰린드롬 모드: PE 리드에서 어댑터가 서로 겹치는 경우 (insert size < read length)
• 단순 모드: 어댑터가 리드 끝에 단순히 붙어있는 경우

SLIDINGWINDOW 상세 매개변수

SLIDINGWINDOW:<windowSize>:<requiredQuality>
  • windowSize: 슬라이딩 윈도우 크기 (bp 단위, 보통 4-5)
  • requiredQuality: 윈도우 내 평균 품질 임계값 (보통 15-20)
🔍 작동 원리:
리드를 5'→3' 방향으로 스캔하면서 지정된 윈도우 크기만큼의 염기들의 평균 품질을 계산합니다.
평균 품질이 임계값 미만이면 해당 지점에서 리드를 자릅니다.
📊 권장 설정:
• SLIDINGWINDOW:4:15 - 일반적인 Illumina 데이터
• SLIDINGWINDOW:5:20 - 높은 품질 요구시
• SLIDINGWINDOW:3:12 - 낮은 품질 데이터 구제시

LEADING/TRAILING 상세 매개변수

LEADING:<quality> TRAILING:<quality>
  • LEADING: 리드 시작부분에서 지정된 품질 미만인 염기들을 제거
  • TRAILING: 리드 끝부분에서 지정된 품질 미만인 염기들을 제거
  • quality: 품질 임계값 (Phred score, 보통 3-5)
⚡ 실행 순서: LEADING → SLIDINGWINDOW → TRAILING → MINLEN

MINLEN/MAXLEN 상세 매개변수

MINLEN:<length> MAXLEN:<length>
  • MINLEN: 최소 허용 리드 길이 (bp 단위)
  • MAXLEN: 최대 허용 리드 길이 (bp 단위, 선택사항)
📏 길이 설정 가이드라인:
• 150bp 리드: MINLEN:36 (원래 길이의 24%)
• 100bp 리드: MINLEN:25 (원래 길이의 25%)
• 75bp 리드: MINLEN:20 (원래 길이의 27%)

고급 필터링 옵션

HEADCROP:<length> CROP:<length> AVGQUAL:<quality> MAXINFO:<targetLength>:<strictness>
  • HEADCROP: 리드 시작부분에서 지정된 길이만큼 무조건 제거
  • CROP: 리드를 지정된 길이로 잘라냄
  • AVGQUAL: 전체 리드의 평균 품질이 임계값 미만이면 제거
  • MAXINFO: 정보량 기반 적응적 품질 트리밍

🎯 실무 필터링 시나리오

데이터 특성에 따른 최적 필터링 전략

고품질 식물 게놈 (콩 예시)

trimmomatic PE -threads 4 \ sample_R1.fastq.gz sample_R2.fastq.gz \ output_R1_paired.fq.gz output_R1_unpaired.fq.gz \ output_R2_paired.fq.gz output_R2_unpaired.fq.gz \ ILLUMINACLIP:TruSeq3-PE.fa:2:30:10 \ LEADING:3 TRAILING:3 \ SLIDINGWINDOW:4:15 \ MINLEN:36
  • 특징: 일반적인 Illumina 150bp PE 데이터
  • 목표: 균형잡힌 품질과 수율
  • 예상 수율: 85-95%

저품질 데이터 구제

trimmomatic PE -threads 4 \ poor_quality_R1.fastq.gz poor_quality_R2.fastq.gz \ rescued_R1_paired.fq.gz rescued_R1_unpaired.fq.gz \ rescued_R2_paired.fq.gz rescued_R2_unpaired.fq.gz \ ILLUMINACLIP:TruSeq3-PE.fa:2:30:10:2:true \ LEADING:5 TRAILING:5 \ SLIDINGWINDOW:3:12 \ AVGQUAL:15 \ MINLEN:30
  • 특징: 품질이 전반적으로 낮은 데이터
  • 목표: 최대한 많은 데이터 구제
  • 예상 수율: 60-80%

고정밀 분석용 (변이 검출)

trimmomatic PE -threads 4 \ sample_R1.fastq.gz sample_R2.fastq.gz \ strict_R1_paired.fq.gz strict_R1_unpaired.fq.gz \ strict_R2_paired.fq.gz strict_R2_unpaired.fq.gz \ ILLUMINACLIP:TruSeq3-PE.fa:2:30:10 \ LEADING:5 TRAILING:5 \ SLIDINGWINDOW:5:20 \ AVGQUAL:25 \ MINLEN:50
  • 특징: 엄격한 품질 기준 적용
  • 목표: 최고 품질 데이터만 선별
  • 예상 수율: 70-85%

오래된 DNA 샘플

trimmomatic PE -threads 4 \ degraded_R1.fastq.gz degraded_R2.fastq.gz \ recovered_R1_paired.fq.gz recovered_R1_unpaired.fq.gz \ recovered_R2_paired.fq.gz recovered_R2_unpaired.fq.gz \ ILLUMINACLIP:TruSeq3-PE.fa:2:30:10:2:true \ HEADCROP:5 \ LEADING:3 TRAILING:3 \ SLIDINGWINDOW:4:12 \ MINLEN:25
  • 특징: DNA 분해로 인한 5' 말단 손상
  • 목표: 손상된 부분 제거 후 활용
  • 예상 수율: 50-70%

📊 필터링 결과 해석 가이드

Trimmomatic 출력 통계 해석

Input Read Pairs: 1000000 Both Surviving: 876543 (87.65%) Forward Only Surviving: 45678 (4.57%) Reverse Only Surviving: 23456 (2.35%) Dropped: 54323 (5.43%)

좋은 결과 지표

  • Both Surviving: 85% 이상
  • Total Surviving: 90% 이상
  • Dropped: 10% 이하
  • Forward/Reverse Only: 각각 5% 이하

문제 징후

  • Both Surviving: 70% 미만
  • Dropped: 20% 이상
  • Forward/Reverse Only: 각각 10% 이상
  • 원인: 과도한 필터링 또는 저품질 데이터
결과 최적화 팁
  • Both Surviving이 낮은 경우: SLIDINGWINDOW나 MINLEN 조건 완화
  • Forward/Reverse Only가 높은 경우: 어댑터 제거 설정 점검
  • Dropped이 높은 경우: AVGQUAL이나 MINLEN 임계값 낮추기
  • 어댑터 오염이 심한 경우: minAdapterLength를 6으로 낮추기

💻 Trimmomatic 실행

1단계: Trimmomatic 설치

# Trimmomatic 설치 (conda 사용) conda install -c bioconda trimmomatic # 어댑터 파일 확인 ls $CONDA_PREFIX/share/trimmomatic/adapters/ # 주요 어댑터 파일들: # TruSeq3-PE.fa : TruSeq3 Paired-End 어댑터 # TruSeq3-SE.fa : TruSeq3 Single-End 어댑터 # NexteraPE-PE.fa : Nextera Paired-End 어댑터

TruSeq3-PE.fa 어댑터 파일 내용

Trimmomatic에서 사용되는 TruSeq3 Paired-End 어댑터 서열 파일의 내용입니다:

>PrefixPE/1 TACACTCTTTCCCTACACGACGCTCTTCCGATCT >PrefixPE/2 GTGACTGGAGTTCAGACGTGTGCTCTTCCGATCT
어댑터 서열 설명
  • PrefixPE/1: Read 1용 어댑터 프리픽스 (5' → 3' 방향)
  • PrefixPE/2: Read 2용 어댑터 프리픽스 (5' → 3' 방향)
  • 팰린드롬 모드: PE 리드가 짧은 insert로 겹칠 때 사용
  • 단순 모드: 리드 끝에 어댑터가 붙어있을 때 사용
  • 서열 길이: 각각 33bp의 어댑터 서열
실무 활용 팁
  • 어댑터 검출: Trimmomatic은 이 두 서열을 참조하여 어댑터를 찾아 제거
  • 팰린드롬 모드: 짧은 insert size로 PE 리드가 겹칠 때 자동 적용
  • 단순 모드: 일반적인 어댑터 제거 시 사용
  • 커스텀 어댑터: 다른 라이브러리 키트 사용 시 해당 어댑터 서열로 교체
  • GitHub 소스: 공식 레포지토리에서 최신 버전 확인

2단계: Paired-End 처리

# Paired-End FASTQ 파일 처리 trimmomatic PE -threads 4 \ ERR3184150_1.fastq.gz ERR3184150_2.fastq.gz \ ERR3184150_1_paired.fastq.gz ERR3184150_1_unpaired.fastq.gz \ ERR3184150_2_paired.fastq.gz ERR3184150_2_unpaired.fastq.gz \ ILLUMINACLIP:TruSeq3-PE.fa:2:30:10 \ LEADING:3 \ TRAILING:3 \ SLIDINGWINDOW:4:15 \ MINLEN:36 # 매개변수 설명: # ILLUMINACLIP:TruSeq3-PE.fa:2:30:10 # - 어댑터 파일:seed_mismatches:palindrome_clip_threshold:simple_clip_threshold # LEADING:3 - 앞쪽에서 Q3 미만 염기 제거 # TRAILING:3 - 뒤쪽에서 Q3 미만 염기 제거 # SLIDINGWINDOW:4:15 - 4bp 윈도우에서 평균 Q15 미만 제거 # MINLEN:36 - 36bp 미만 리드 제거

📊 품질 관리 워크플로우

효율적인 품질 관리를 위한 단계별 워크플로우입니다.

1. 원본 데이터 확인

FASTQ 파일 무결성 검사 및 기본 정보 확인

2. FastQC 품질 평가

원본 데이터의 품질 지표 분석 및 문제점 식별

3. Trimmomatic 전처리

어댑터 제거, 품질 필터링, 길이 필터링 수행

4. 처리 후 품질 검사

전처리된 데이터에 대한 FastQC 재분석

5. 다음 단계 진행

품질이 확보된 데이터로 게놈 정렬 단계 진행

🎯 품질 관리 체크포인트

최종 확인 사항

처리 전 vs 후

  • 평균 품질: Q30 이상 유지
  • 어댑터 함량: 0% 또는 미검출
  • 리드 길이: 일정한 분포
  • 데이터 손실: 20% 이하

통계 확인

  • 총 리드 수: 분석에 충분한 양
  • 페어링 비율: 90% 이상
  • GC 함량: 정상 범위
  • 중복도: 허용 범위 내
