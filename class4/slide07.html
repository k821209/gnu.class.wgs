⚙️ 1단계: GATK 환경 설정

GATK 설치 확인

GATK이 올바르게 설치되어 있는지 확인하고 버전을 체크합니다.

# GATK 버전 확인 gatk --version # 또는 Java 기반 실행 java -jar /opt/gatk/gatk-package-4.3.0.0-local.jar --version # 도움말 확인 gatk --help

참조 게놈 인덱스 준비

GATK에서 사용할 참조 게놈 인덱스를 생성합니다.

# 작업 디렉토리 이동 cd /home/ubuntu/edu/reference # FASTA 인덱스 생성 (이미 있으면 생략) samtools faidx soybean_genome.fa # GATK 딕셔너리 생성 gatk CreateSequenceDictionary \ -R soybean_genome.fa \ -O soybean_genome.dict # 생성된 파일 확인 ls -la soybean_genome.fa.* ls -la soybean_genome.dict
soybean_genome.fa soybean_genome.fa.fai soybean_genome.dict

🧬 2단계: HaplotypeCaller 실행

gVCF 모드로 변이 호출

개별 샘플의 gVCF를 생성합니다. 이는 나중에 다중 샘플 분석에 유용합니다.

# 결과 디렉토리 생성 cd /home/ubuntu/edu/ mkdir -p variant_calling # GATK HaplotypeCaller 실행 (gVCF 모드) gatk HaplotypeCaller \ -R /home/ubuntu/edu/reference/soybean_genome.fa \ -I alignment_results/soybean_sorted.bam \ -O variant_calling/soybean_raw.g.vcf.gz \ -ERC GVCF \ --native-pair-hmm-threads 4 # 진행 상황 확인 ls -lh variant_calling/

GATK HaplotypeCaller 주요 옵션

필수 매개변수

  • -R: 참조 게놈 FASTA 파일
  • -I: 입력 BAM 파일
  • -O: 출력 VCF/gVCF 파일
  • -ERC GVCF: gVCF 모드 활성화

성능 옵션

  • --native-pair-hmm-threads: 스레드 수
  • 샘플 이름: BAM 파일의 Read Group에서 자동 추출
  • -L: 특정 영역만 분석
  • --tmp-dir: 임시 디렉토리 지정

🔄 3단계: GenotypeGVCFs 실행

gVCF를 최종 VCF로 변환

gVCF 파일을 최종 분석 가능한 VCF 파일로 변환합니다.

# GenotypeGVCFs 실행 gatk GenotypeGVCFs \ -R /home/ubuntu/edu/reference/soybean_genome.fa \ -V variant_calling/soybean_raw.g.vcf.gz \ -O variant_calling/soybean_gatk.vcf.gz # VCF 인덱스 생성 gatk IndexFeatureFile \ -I variant_calling/soybean_gatk.vcf.gz # 파일 크기 확인 ls -lh variant_calling/*.vcf*

🛠️ 대안: bcftools를 이용한 변이 호출

bcftools mpileup + call 파이프라인

GATK 대신 bcftools를 사용한 변이 호출 방법입니다. 더 빠르고 메모리 효율적입니다.

# bcftools mpileup으로 변이 호출 bcftools mpileup \ -f /home/ubuntu/edu/reference/soybean_genome.fa \ -q 20 \ -Q 20 \ -a FORMAT/DP,FORMAT/AD \ alignment_results/soybean_sorted.bam | \ bcftools call \ -mv \ -Oz \ -o variant_calling/soybean_bcftools.vcf.gz # VCF 인덱스 생성 bcftools index variant_calling/soybean_bcftools.vcf.gz # 변이 정규화 (선택사항) bcftools norm \ -f /home/ubuntu/edu/reference/soybean_genome.fa \ -m -any \ variant_calling/soybean_bcftools.vcf.gz \ -Oz -o variant_calling/soybean_bcftools_norm.vcf.gz bcftools index variant_calling/soybean_bcftools_norm.vcf.gz

GATK vs bcftools 비교

GATK HaplotypeCaller

  • 장점: 높은 정확도, gVCF 지원
  • 단점: 높은 메모리 사용량, 느림
  • 적합: 고품질 분석, 다중 샘플
  • 특징: 로컬 어셈블리 기반

bcftools mpileup

  • 장점: 빠른 속도, 낮은 메모리 사용
  • 단점: 상대적으로 낮은 정확도
  • 적합: 빠른 스크리닝, 대용량 데이터
  • 특징: 통계적 모델 기반

bcftools 상세 매개변수

mpileup 옵션

  • -f: 참조 게놈 FASTA 파일
  • -q: 최소 매핑 품질 (기본값: 0)
  • -Q: 최소 염기 품질 (기본값: 13)
  • -a: 추가 주석 정보
  • -d: 최대 깊이 (기본값: 250)

call 옵션

  • -m: 다중 대립유전자 모델
  • -v: 변이만 출력
  • -Oz: gzip 압축 출력
  • -o: 출력 파일명
  • -c: 컨센서스 모델 (대안)

VCF 파일 미리보기 및 비교

# GATK 결과 확인 echo "=== GATK 결과 ===" bcftools view -h variant_calling/soybean_gatk.vcf.gz | head -20 bcftools view -H variant_calling/soybean_gatk.vcf.gz | head -5 echo "=== bcftools 결과 ===" bcftools view -h variant_calling/soybean_bcftools.vcf.gz | head -20 bcftools view -H variant_calling/soybean_bcftools.vcf.gz | head -5 # 변이 개수 비교 echo "=== 변이 개수 비교 ===" echo "GATK 변이 수:" bcftools stats variant_calling/soybean_gatk.vcf.gz | grep "^SN" echo "bcftools 변이 수:" bcftools stats variant_calling/soybean_bcftools.vcf.gz | grep "^SN"

📊 4단계: 변이 통계 분석

Python을 이용한 변이 분석

# !pip install koreanize-matplotlib 터미널에 입력하여 설치 # 설치 후 import로 불러오기 import pandas as pd import matplotlib.pyplot as plt import seaborn as sns import subprocess import numpy as np from collections import Counter import gzip import koreanize_matplotlib def analyze_vcf_statistics(vcf_file): """VCF 파일의 변이 통계를 분석합니다.""" # bcftools stats 실행 result = subprocess.run(['bcftools', 'stats', vcf_file], capture_output=True, text=True) stats = {} lines = result.stdout.strip().split('\n') for line in lines: if line.startswith('SN'): parts = line.split('\t') if len(parts) >= 4: key = parts[2].strip(':') value = parts[3] stats[key] = value return stats def parse_vcf_variants(vcf_file): """VCF 파일에서 변이 정보를 파싱합니다.""" variants = [] # gzip 파일인지 확인 if vcf_file.endswith('.gz'): file_opener = gzip.open mode = 'rt' else: file_opener = open mode = 'r' with file_opener(vcf_file, mode) as f: for line in f: if line.startswith('#'): continue fields = line.strip().split('\t') if len(fields) < 8: continue chrom = fields[0] pos = int(fields[1]) ref = fields[3] alt = fields[4] qual = float(fields[5]) if fields[5] != '.' else 0 info = fields[7] # 변이 유형 결정 if len(ref) == 1 and len(alt) == 1: variant_type = 'SNP' elif len(ref) > len(alt): variant_type = 'Deletion' elif len(ref) < len(alt): variant_type = 'Insertion' else: variant_type = 'Complex' # DP 정보 추출 dp = 0 for info_field in info.split(';'): if info_field.startswith('DP='): dp = int(info_field.split('=')[1]) break variants.append({ 'chrom': chrom, 'pos': pos, 'ref': ref, 'alt': alt, 'qual': qual, 'type': variant_type, 'depth': dp }) return pd.DataFrame(variants) def plot_variant_analysis(df): """변이 분석 결과를 시각화합니다.""" fig, axes = plt.subplots(2, 2, figsize=(15, 12)) # 변이 유형별 개수 type_counts = df['type'].value_counts() axes[0, 0].pie(type_counts.values, labels=type_counts.index, autopct='%1.1f%%') axes[0, 0].set_title('변이 유형별 분포') # 염색체별 변이 개수 chrom_counts = df['chrom'].value_counts().head(10) axes[0, 1].bar(range(len(chrom_counts)), chrom_counts.values) axes[0, 1].set_xticks(range(len(chrom_counts))) axes[0, 1].set_xticklabels(chrom_counts.index, rotation=45) axes[0, 1].set_title('염색체별 변이 개수 (상위 10개)') axes[0, 1].set_ylabel('변이 개수') # 품질 점수 분포 axes[1, 0].hist(df[df['qual'] > 0]['qual'], bins=50, alpha=0.7) axes[1, 0].set_title('변이 품질 점수 분포') axes[1, 0].set_xlabel('QUAL Score') axes[1, 0].set_ylabel('빈도') axes[1, 0].axvline(x=30, color='red', linestyle='--', label='품질 임계값 (Q30)') axes[1, 0].legend() # 깊이 분포 depth_data = df[df['depth'] > 0]['depth'] axes[1, 1].hist(depth_data, bins=50, alpha=0.7) axes[1, 1].set_title('변이 깊이 분포') axes[1, 1].set_xlabel('Read Depth') axes[1, 1].set_ylabel('빈도') axes[1, 1].axvline(x=depth_data.mean(), color='red', linestyle='--', label=f'평균 깊이: {depth_data.mean():.1f}') axes[1, 1].legend() plt.tight_layout() plt.savefig('variant_analysis.png', dpi=300, bbox_inches='tight') plt.show() def compare_variant_callers(gatk_file, bcftools_file): """두 변이 호출 방법을 비교합니다.""" print("🔬 GATK vs bcftools 변이 호출 비교") print("=" * 60) # 각각 분석 gatk_stats = analyze_vcf_statistics(gatk_file) bcftools_stats = analyze_vcf_statistics(bcftools_file) gatk_df = parse_vcf_variants(gatk_file) bcftools_df = parse_vcf_variants(bcftools_file) # 비교 표 생성 comparison_data = { 'Method': ['GATK', 'bcftools'], 'Total_Variants': [len(gatk_df), len(bcftools_df)], 'SNPs': [len(gatk_df[gatk_df['type'] == 'SNP']), len(bcftools_df[bcftools_df['type'] == 'SNP'])], 'INDELs': [len(gatk_df[gatk_df['type'] != 'SNP']), len(bcftools_df[bcftools_df['type'] != 'SNP'])], 'Avg_Quality': [gatk_df[gatk_df['qual'] > 0]['qual'].mean(), bcftools_df[bcftools_df['qual'] > 0]['qual'].mean()], 'Avg_Depth': [gatk_df[gatk_df['depth'] > 0]['depth'].mean(), bcftools_df[bcftools_df['depth'] > 0]['depth'].mean()] } comp_df = pd.DataFrame(comparison_data) print("\n📊 변이 호출 방법 비교:") print(comp_df.to_string(index=False)) # 겹치는 변이 분석 gatk_positions = set(zip(gatk_df['chrom'], gatk_df['pos'])) bcftools_positions = set(zip(bcftools_df['chrom'], bcftools_df['pos'])) overlap = len(gatk_positions & bcftools_positions) gatk_only = len(gatk_positions - bcftools_positions) bcftools_only = len(bcftools_positions - gatk_positions) print(f"\n🎯 변이 위치 겹침 분석:") print(f"공통 변이: {overlap:,}") print(f"GATK 전용: {gatk_only:,}") print(f"bcftools 전용: {bcftools_only:,}") print(f"겹침률: {overlap/len(gatk_positions | bcftools_positions)*100:.1f}%") return comp_df # 분석 실행 gatk_file = '/home/ubuntu/edu/raw_data/variant_calling/soybean_gatk.vcf.gz' bcftools_file = '/home/ubuntu/edu/raw_data/variant_calling/soybean_bcftools.vcf.gz' # 개별 분석 print("🧬 GATK 변이 호출 통계:") print("=" * 50) gatk_stats = analyze_vcf_statistics(gatk_file) for key, value in gatk_stats.items(): if 'number of' in key.lower() or 'SNPs' in key or 'indels' in key: print(f"{key}: {value}") gatk_df = parse_vcf_variants(gatk_file) print(f"\n📊 GATK 상세 분석:") print(f"총 변이 수: {len(gatk_df):,}") print(f"SNP: {len(gatk_df[gatk_df['type'] == 'SNP']):,}") print(f"Insertion: {len(gatk_df[gatk_df['type'] == 'Insertion']):,}") print(f"Deletion: {len(gatk_df[gatk_df['type'] == 'Deletion']):,}") print(f"평균 품질 점수: {gatk_df[gatk_df['qual'] > 0]['qual'].mean():.2f}") print(f"평균 깊이: {gatk_df[gatk_df['depth'] > 0]['depth'].mean():.1f}") # 비교 분석 comparison_results = compare_variant_callers(gatk_file, bcftools_file) # 시각화 (GATK 결과) plot_variant_analysis(gatk_df)

🏷️ 5단계: 변이 주석 (SnpEff)

변이 기능 예측 및 주석

SnpEff를 사용하여 변이가 유전자에 미치는 영향을 예측합니다.

# SnpEff 데이터베이스 확인 (콩 게놈) snpEff databases | grep -i glycine # 변이 주석 실행 (예시: 일반적인 식물 데이터베이스 사용) snpEff -v Glycine_max \ variant_calling/soybean_raw.vcf.gz > \ variant_calling/soybean_annotated.vcf # 주석 통계 확인 snpEff -stats variant_calling/snpEff_summary.html \ Glycine_max \ variant_calling/soybean_raw.vcf.gz > \ variant_calling/soybean_annotated.vcf

변이 기능 분류

높은 영향 (HIGH)

  • STOP_GAINED: 조기 종료 코돈
  • STOP_LOST: 종료 코돈 손실
  • START_LOST: 시작 코돈 손실
  • FRAME_SHIFT: 프레임시프트 돌연변이

중간 영향 (MODERATE)

  • MISSENSE: 아미노산 변화
  • CODON_DELETION: 코돈 결실
  • CODON_INSERTION: 코돈 삽입
  • SPLICE_SITE: 스플라이싱 부위

식물 게놈 변이 호출 특성

  • SNP 밀도: 콩 게놈에서 약 1-5 SNP/kb 예상
  • INDEL 비율: 전체 변이의 15-25% 정도
  • Ti/Tv 비율: Transition/Transversion 약 2.0-2.5
  • 이형접합도: 자가수분 식물에서 상대적으로 낮음

주의사항

  • 메모리 사용량: GATK는 대용량 메모리 필요 (8GB 이상 권장)
  • 처리 시간: 게놈 크기에 따라 수시간 소요 가능
  • 임시 파일: 충분한 디스크 공간 확보 필요
  • 참조 게놈 품질: 높은 품질의 참조 게놈 사용 중요
  • BAM 품질: 정렬 품질이 변이 호출 정확도에 직접 영향
